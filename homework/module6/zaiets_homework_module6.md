## Код скрипта (`backup.sh`)

```bash
#!/bin/bash

# 1. Перевірка аргументів (має бути 2 аргументи і обидва мають бути існуючими каталогами)
if [ "$#" -ne 2 ] || [ ! -d "$1" ] || [ ! -d "$2" ]; then
    echo "Usage: ./backup.sh <log_dir> <backup_dir>"
    exit 1
fi

LOG_DIR="$1"
BACKUP_DIR="$2"
LOCK_FILE="/tmp/backup.lock"

# 2. Захист від паралельного запуску
if [ -f "$LOCK_FILE" ]; then
    echo "Backup already running"
    exit 1
fi

# Створюємо lock-файл для блокування паралельних запусків
touch "$LOCK_FILE"

# Гарантуємо видалення lock-файлу при завершенні скрипта (навіть якщо він впаде з помилкою)
trap "rm -f $LOCK_FILE" EXIT

# 3. Формування імені архіву з поточною датою та часом
TIMESTAMP=$(date +"%Y-%m-%d_%H-%M")
ARCHIVE_NAME="logs_backup_${TIMESTAMP}.tar.gz"
ARCHIVE_PATH="${BACKUP_DIR}/${ARCHIVE_NAME}"

# 4. Створення архіву логів та перевірка результату
if tar -czf "$ARCHIVE_PATH" -C "$LOG_DIR" . 2>/dev/null; then
    echo "Backup created: $ARCHIVE_PATH"
else
    echo "Backup failed"
    exit 2
fi

Короткий опис роботи скрипта
Цей Bash-скрипт автоматизує створення резервних копій лог-файлів.
Спочатку він перевіряє, чи передано рівно два параметри і чи існують відповідні директорії. Далі скрипт створює файл блокування (/tmp/backup.lock), щоб уникнути конфліктів, якщо хтось спробує запустити бекап двічі одночасно. Після цього за допомогою утиліти tar створюється архів усіх файлів із папки логів, а в назву архіву автоматично підставляється поточна дата та час. Якщо архівація проходить успішно, скрипт виводить повний шлях до створеного бекапу, якщо ж стається помилка — завершує роботу з відповідним повідомленням і кодом 2. Завдяки команді trap файл блокування надійно видаляється по завершенню роботи.
